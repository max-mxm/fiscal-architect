# Documentation — Fiscal Architect

Documentation vivante du projet Fiscal Architect. Chaque fichier a un rôle précis et doit être maintenu à jour au fil du développement.

## Fichiers

| Fichier | Description |
|---------|-------------|
| [objectifs.md](./objectifs.md) | Milestones M0-M5 et vision produit |
| [taches.md](./taches.md) | Liste exhaustive des tâches avec identifiants et statuts |
| [journal.md](./journal.md) | Log de développement (problèmes, décisions, solutions) |
| [architecture.md](./architecture.md) | Décisions d'architecture (ADR) |
| [progression.md](./progression.md) | Tableau de progression par milestone |

## Conventions

- **Statuts des tâches** : `[ ]` todo · `[~]` en cours · `[x]` fait · `[!]` bloqué
- **Identifiants** : chaque tâche a un ID unique (ex: `DASH-01`, `FOUND-02`) utilisable dans les commits
- **Commits** : format `TASK-ID: description` (ex: `FOUND-01: add useLocalStorage hook`)
- **Mise à jour** : toujours mettre à jour `taches.md` et `progression.md` après chaque tâche terminée
